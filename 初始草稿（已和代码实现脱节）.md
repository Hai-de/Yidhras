这个平台包含叙事社交系统 + 情报系统 + 关系可视化系统，核心功能将是通过处理延迟信息、分析复杂关系网来操纵代理人（Agents），平台的外表将会是一个社交外壳，最左边将是discord和vscode一样的侧边栏，随后是左中右三大块分割

1) Agent Inference Interface（推理接口）
- 难点不是“接 LLM API”，而是策略稳定性：提示词版本管理、失败重试、幂等、成本控制、幻觉约束。
- 当前正式路线不再是一次性接口验证，而是先做 D-ready 的 Phase B，再进入持久化工作流 Phase D。
- Phase B 要先冻结上下文、prompt、decision、intent draft 等契约，避免后续大规模返工。
- Phase D 要显式承担 trace / workflow / retry / replay / audit 的系统复杂度。

2) Memory Core（长期记忆/RAG）
- 难点在记忆写入与检索策略，不在存储本身。
- 什么该记、何时遗忘、冲突事实如何合并、叙事时序如何回放，这些是产品逻辑决策，代码只是执行器。
- Memory Core 应建立在稳定的 trace / decision / intent schema 之上，而不是抢先绑定临时推理输出。

3) L4 传输层 + 跨层级联动（L1-L4）
- 这是系统最有价值也最复杂的部分：延迟/丢包规则会反过来影响叙事与关系演化。
- 我能实现事件调度与仿真引擎，但参数校准（“像不像你想要的世界”）必须依赖你给的设计目标。
- Action Dispatcher 最终应消费持久化后的 ActionIntent，而不是直接耦合推理输出。

#### 新节点
中介节点 (Relay/Intermediary Node)：负责具象化 L4 的传输与资源，纯代码驱动的被动节点。它代表了情报传递的链路（如：基站、信使、论坛板块、被利用的资源）。性能与生命周期机制： 为避免图谱无限膨胀，它遵循严格的回收机制——除非被使用者手动 Pin 住（钉死作为重要线索），否则一旦其拓扑路径无法触达任何“活跃 节点”，将被后端/前端清理。原来活跃子节点创建的子节点归为现在的中介节点，毕竟子节点也是一种资源


容器节点 (Container/Unresolved Node)：情报分析中的迷雾与系统边界托底，一种预留的、状态为空的节点。视觉托底： 如果后端未来引入了前端尚未支持的 ActionIntent 类型，前端不再抛错，而是将其渲染为统一的“黑盒容器”，保证图谱的鲁棒性。叙事价值： 它可以代表情报学中的“未决实体（Unresolved Entity）”。当发生了一次匿名动作（Social Noise），或者推理解析失败时，它在画布上作为一个“未知源”，等待未来被合并、揭面或妥善封存。
#### 功能
网状图谱: Obsidian Graph View，融入Notion 的块理念，当通过过滤条件选中特定节点的局部关系图的时候能显现特定根节点下面的各种子节点标签
树状图谱: 家族族谱，参考Palantir Gotham / Maltego 的树状布局，具体身份和活跃节点分离，这样适用于公司和等级社会
