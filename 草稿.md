我需要一个前后端分离的跨平台网页，技术栈我打算暂时使用node.js作为后端，前端是使用Vue.js的nuxt，所有的数据保存将会在本地data文件夹里面
有插件系统用于将来拓展，有可供导入和导出的世界观包来设定整体提示词环境
在这个平台上活动的节点是由ai控制的Agents，这意味着提示词编排工具也将要设计
这个平台包含叙事社交系统 + 情报系统 + 关系可视化系统，核心功能将是通过处理延迟信息、分析复杂关系网来操纵代理人（Agents），平台的外表将会是一个社交外壳，最左边将是discord和vscode一样的侧边栏，随后是左中右三大块分割
侧边栏的一些按钮目前我的打算是有以下功能:
- 如同“X/Twitter”的信息流作为强“公开舆论场”用来容纳:"公共动态/热点传播/转发链/影响力扩散/节点之间的舆论波及"
- 第二个按钮用来容纳关系图谱，关系图谱里面将来会容纳有向图、人物关系网、影响路径来表达血缘关系 / 政治关系 / 金融关系 / 情报关系，这意味这关系图谱里面会有二级菜单来区分结构化的关系比如族谱或者公司职位架构等代表两个对象关系的内容，也会有非结构化的展示如同Obsidian等关系图
- 第三个按钮里面的是内容档案，用于存放和查找对象的档案和事件的日志以及历史时间线

这个平台将会按照四层系统来设计
第一层社交层是像Discord / X 展示：
- 私聊
- 群聊
- 广播
- 舆论 feed
- 评论/转发/引用
- agent 发言
第二层关系层像 Obsidian / Kumu展示：
- 人物关系
- 势力关系
- 血缘关系
- 指挥链
- 利益链
- 感染链
- 信息传播路径
但这些在抽象中本质上都是节点和线的关系
第三层叙事层应该像时间线工具

展示：
- 事件日志
- 世界历史
- 战争进程
- 家族兴衰
- 任务链
- 情报档案
第四层传输层比较特殊，关注的是节点和对象之间连接的线的影响
用于展示：
- 信息传递方式
- ETA 到达时间
- 中继节点
- 拦截风险
- 噪声/失真率
- 加密/破译
- 生物扩散 / 星际延迟 / 信鸽投递状态

1) Agent Inference Interface（推理接口）
- 难点不是“接 LLM API”，而是策略稳定性：提示词版本管理、失败重试、幂等、成本控制、幻觉约束。
- 当前正式路线不再是一次性接口验证，而是先做 D-ready 的 Phase B，再进入持久化工作流 Phase D。
- Phase B 要先冻结上下文、prompt、decision、intent draft 等契约，避免后续大规模返工。
- Phase D 要显式承担 trace / workflow / retry / replay / audit 的系统复杂度。

2) Memory Core（长期记忆/RAG）
- 难点在记忆写入与检索策略，不在存储本身。
- 什么该记、何时遗忘、冲突事实如何合并、叙事时序如何回放，这些是产品逻辑决策，代码只是执行器。
- Memory Core 应建立在稳定的 trace / decision / intent schema 之上，而不是抢先绑定临时推理输出。

3) L4 传输层 + 跨层级联动（L1-L4）
- 这是系统最有价值也最复杂的部分：延迟/丢包规则会反过来影响叙事与关系演化。
- 我能实现事件调度与仿真引擎，但参数校准（“像不像你想要的世界”）必须依赖你给的设计目标。
- Action Dispatcher 最终应消费持久化后的 ActionIntent，而不是直接耦合推理输出。
