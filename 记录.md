## Runtime & Acceptance Notes / 运行时与验收边界 (2026-03-21)

### Accepted Constraints / 已冻结边界

- Degraded startup is acceptable for fresh clones when `health_level=degraded` and `runtime_ready=false`.
- 首次拉取仓库时，若 `health_level=degraded` 且 `runtime_ready=false`，视为允许启动。
- World-pack-gated endpoints must consistently return `503/WORLD_PACK_NOT_READY` when runtime is not ready.
- 受 world-pack 约束的接口在运行时未就绪时必须统一返回 `503/WORLD_PACK_NOT_READY`。
- `prepare:runtime` is the single startup preparation entry.
- `prepare:runtime` 是统一启动前置准备入口。
- Runtime speed policy follows `override > world_pack > default(1)`.
- 运行时速度策略遵循 `override > world_pack > default(1)`。

### Delivered Follow-Ups / 已落地跟进

- Runtime speed override API: `POST /api/runtime/speed`
- `runtime_speed.override_since` added to status/override snapshots
- Notification `details` enriched with runtime-speed metadata
