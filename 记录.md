## Next-Stage Candidate Rules (Warn First) / 下一阶段候选规则（先警告）

Purpose / 用途:

- These rules are candidates for pre-kickoff evaluation.
- 这些规则用于开工前评估，不在当前阶段直接收紧为 error。

### Shared Candidates (Server + Web) / 前后端通用候选

- `@typescript-eslint/consistent-type-imports`: `warn`
  - Why: keeps type-only imports explicit and reduces runtime-import confusion.
  - 理由：明确类型导入，降低运行时导入歧义。
- `eqeqeq`: `warn`
  - Why: avoids loose equality edge cases.
  - 理由：避免宽松比较带来的边界问题。
- `curly`: `warn`
  - Why: prevents accidental single-line branch bugs.
  - 理由：减少单行分支引发的误判和维护风险。

### Server-Focused Candidates / 后端候选

- `@typescript-eslint/no-floating-promises`: `warn` (evaluate runtime noise first)
  - Why: helps catch unhandled async flows in API/service paths.
  - 理由：帮助发现 API/服务链路中未处理的异步调用。
- `@typescript-eslint/await-thenable`: `warn`
  - Why: catches invalid `await` usage.
  - 理由：识别错误的 `await` 用法。

### Web-Focused Candidates / 前端候选

- `vue/no-mutating-props`: `warn`
  - Why: avoids hidden state coupling in component tree.
  - 理由：避免组件树中的隐性状态耦合。
- `vue/no-setup-props-destructure`: `warn` (only if team wants conservative reactivity safety)
  - Why: reduces accidental reactivity loss in setup usage.
  - 理由：降低 setup 中解构 props 导致响应性丢失的风险。

### Not Recommended to Dual-Enable / 不建议重复启用

- `import/order` is not recommended while using `simple-import-sort`.
- 在已启用 `simple-import-sort` 的前提下，不建议再启用 `import/order`，避免规则冲突。
