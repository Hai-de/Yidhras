## 4. 测试模式

### 4.1 路由层测试模式
```typescript
import { createTestApp } from '../../helpers/test_app.js';
import { createMockAppContext } from '../../helpers/mock_context.js';

describe('route tests', () => {
  it('handles request', async () => {
    const ctx = createMockAppContext();
    const app = createTestApp(ctx, { operator: { id: 'op-1', is_root: true } });
    someRoutes.register(app.express, ctx);
    const res = await app.get('/api/endpoint');
    expect(res.status).toBe(200);
    await app.close();
  });
});
```

### 4.2 服务层测试模式
```typescript
import { createMockAppContext } from '../../helpers/mock_context.js';

describe('service tests', () => {
  it('processes request', async () => {
    const ctx = createMockAppContext();
    ctx.prisma.someModel.findMany.mockResolvedValue([mockData]);
    const result = await someService(ctx, 'id');
    expect(result).toBeDefined();
  });
});
```

### 4.3 纯函数测试模式
```typescript
describe('pure function tests', () => {
  it('transforms input', () => {
    const result = pureFunction(input);
    expect(result).toEqual(expected);
```

## 7. 风险与缓解

### 7.1 技术风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 复杂依赖mock | 测试难以编写 | 使用依赖注入和mock框架 |
| 测试执行时间长 | 开发效率低 | 并行测试和选择性测试 |
| 测试数据库隔离 | 测试不稳定 | 事务回滚和数据库清理 |

### 7.2 进度风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 覆盖率提升缓慢 | 项目延期 | 调整优先级，专注高影响模块 |
| 测试质量不高 | 维护成本高 | 加强代码审查和质量指标 |
| 资源不足 | 进度延迟 | 适当延长阶段时间 |
